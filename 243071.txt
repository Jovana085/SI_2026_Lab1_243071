22. Да, настана конфликт при спојување на гранката master во feature-borrow-book.
Конфликтот беше решен рачно со уредување на датотеката SI2026Lab1Main.java, при што беше избрана една од понудените верзии и беа отстранети конфликтните ознаки (<<<<<<<, =======, >>>>>>>). Беше употребена recursive merge стратегија.
Излез од git status по завршување на merge-от:

On branch feature-borrow-book
nothing to commit, working tree clean
23. Тип на merge: Fast-forward
По ажурирање на гранката feature-genre-report со master, спојувањето можеше да се изврши како fast-forward бидејќи немаше дополнителни commits во master.

24. Тип на merge: Recursive merge
Merge-от не беше fast-forward бидејќи и двете гранки (master и feature-borrow-book) имаа сопствени commits, па Git креираше merge commit.

git commit -m "Final commit"
27. git push origin master - Ова го испраќа финалниот commit заедно со logs.txt и 243071.txt на оддалечениот репозиториум.
